# Hiragen Deployment Guide

## Architecture

- **API** (Express + Prisma) ‚Üí Render.com (Docker)
- **Web** (Next.js 14) ‚Üí Vercel
- **Database** ‚Üí PostgreSQL (Neon, Supabase, or Render managed DB)

---

## Prerequisites

1. A PostgreSQL database (recommended: [Neon](https://neon.tech) free tier or Render managed PostgreSQL)
2. A [Render.com](https://render.com) account
3. A [Vercel](https://vercel.com) account
4. API keys: `GROQ_API_KEY` and/or `GEMINI_API_KEY`

---

## 1. Deploy the API to Render

### Option A: Render Blueprint (recommended)

1. Push your code to GitHub
2. Go to [Render Dashboard](https://dashboard.render.com) ‚Üí **New** ‚Üí **Blueprint**
3. Connect your repo, point to `apps/api/render.yaml`
4. Fill in the environment variables marked `sync: false`:
   - `DATABASE_URL` ‚Äî your PostgreSQL connection string
   - `GROQ_API_KEY` ‚Äî your Groq API key
   - `GEMINI_API_KEY` ‚Äî your Gemini API key (optional fallback)
5. Deploy

### Option B: Manual Docker deploy

1. Go to Render Dashboard ‚Üí **New** ‚Üí **Web Service**
2. Connect your repo
3. Set **Root Directory** to `apps/api`
4. Set **Environment** to **Docker**
5. Add environment variables:
   - `DATABASE_URL` = your PostgreSQL connection string
   - `JWT_SECRET` = a strong random string (64+ chars)
   - `FRONTEND_URL` = `https://hiragen.vercel.app` (your Vercel domain)
   - `GROQ_API_KEY` = your Groq key
   - `GEMINI_API_KEY` = your Gemini key (optional)
   - `ENCRYPTION_KEY` = a strong random string
   - `NODE_ENV` = `production`
6. Deploy

### Post-deploy

- Verify health: `curl https://your-api.onrender.com/api/health`
- Seed the database (one-time): Connect to the service shell and run `npx tsx prisma/seed.ts`

---

## 2. Deploy the Web App to Vercel

1. Go to [Vercel](https://vercel.com) ‚Üí **Add New Project**
2. Import your GitHub repo
3. Set **Root Directory** to `apps/web`
4. Set **Framework Preset** to **Next.js**
5. Add environment variables:
   - `NEXT_PUBLIC_API_URL` = `https://your-api.onrender.com` (your Render API URL)
   - `NEXT_PUBLIC_ESCROW_CONTRACT_ADDRESS` = your deployed contract address
6. Deploy

---

## 3. Post-Deployment Checklist

- [ ] API health check returns `{ "status": "ok" }`
- [ ] CORS: `FRONTEND_URL` on API matches your Vercel domain exactly
- [ ] Database migrations applied (the Docker CMD runs `prisma db push` on startup)
- [ ] Seed data loaded if needed
- [ ] Agent worker is polling (check API logs for "Agent worker started")
- [ ] Test auth flow: register ‚Üí login ‚Üí create task
- [ ] Test agent execution: create a task and verify bot agents pick it up

## Environment Variables Reference

### API (Render)

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `JWT_SECRET` | Yes | Secret for JWT signing (auto-generated by Render blueprint) |
| `FRONTEND_URL` | Yes | Vercel app URL for CORS |
| `GROQ_API_KEY` | Yes | Groq API key for LLM |
| `GEMINI_API_KEY` | No | Fallback LLM provider |
| `ENCRYPTION_KEY` | Yes | For encrypting stored credentials |
| `NODE_ENV` | Yes | `production` |
| `PORT` | No | Defaults to `4000` |

### Web (Vercel)

| Variable | Required | Description |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | Yes | Full URL of the deployed API |
| `NEXT_PUBLIC_ESCROW_CONTRACT_ADDRESS` | Yes | Deployed escrow contract address |

---

## Troubleshooting

- **Prisma errors on startup**: Ensure `DATABASE_URL` is correct and the database is accessible from Render's network.
- **CORS errors**: Make sure `FRONTEND_URL` exactly matches your Vercel domain (including `https://`, no trailing slash).
- **Build failures on Render**: Check that the Dockerfile context is `apps/api` (the root directory setting).
- **Vercel build failures**: Ensure the root directory is set to `apps/web` so it finds `next.config.js`.
